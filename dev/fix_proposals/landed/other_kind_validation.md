# "other" AB kind drill — validation corpus (478 bundles)

**Date:**
**Scope:** 478 bundles classified as `other` in `/tmp/val-verify.json`
(validation corpus, 302 entities, windows target).
**Compare:** confirms / scales up the test-corpus drill (66 bundles) in
`other_kind_drill.md`.

## TL;DR

All 478 "other" bundles share the **same single class signature**:
`{Texture2D(28), AssetBundle(142)}`. They are legacy `Qm…`-prefix CIDv0
standalone-textures misclassified by `abgen-verify`'s `kind_of` dispatch
because `emits_metadata_textasset` suppresses the metadata TextAsset for
CIDv0 hashes (`builder.rs:103-108`) and the `standalone-texture` branch
requires TextAsset(49) (`abgen-verify.rs:182-185`).

No new AB kind — the right fix is a classifier relabel
(**`standalone-texture-legacy`**, proposed in `other_kind_drill.md` P1).

## 2. Reclassification proposal

### P1 — Promote signature `{Texture2D, AssetBundle}` → `standalone-texture-legacy`

Diff in `abgen-verify.rs:kind_of` (3 lines, attribution-only — zero
builder change):

```rust
// inserted after the existing standalone-texture check
if !has(C_GO) && !has(C_TRANSFORM)
    && has(C_TEXTURE2D) && has(C_ASSETBUNDLE)
    && only_in(&[C_TEXTURE2D, C_ASSETBUNDLE]) {
    return "standalone-texture-legacy";
}
```

Effect on the per-kind summary:

```
before: other 478 23 231 182 494265704 235965036 477405.2
after: standalone-texture-legacy 478 23 231 182 494265704 235965036 477405.2
         (other row disappears — no remaining bundles match the fallback)
```

This is the only signature in the bucket, so no other promotion is
warranted.

### P2 — Optional: collapse legacy into `standalone-texture`

Alternative: drop the TextAsset(49) requirement from the existing
`standalone-texture` branch instead of adding a new kind. This would
merge the 478 legacy bundles into the same row as the 208 modern
`standalone-texture` (bafkrei*) bundles, recomputing that row as:

```
standalone-texture-modern 208 17... ppm ≈ 280K (estimated)
standalone-texture-all 686 40... ppm ≈ 360K
```

P1 (separate kind) is preferred — the two cohorts have very different
byte/bit profiles (.resS-streamed share, byte-ident rate) and merging
hides that signal.

## 3. Recovery estimate (if reclassified)

**Zero bytes recoverable from the reclassification itself.** The
classifier change is pure attribution. The actual bit/byte recovery
work is already owned by existing tasks:

| Driver                                | Owning work     | Share of 235M bits | Expected outcome |
|---------------------------------------|-----------------|--------------------|-------------------|
| `.resS` LZ4HC chunk-boundary drift    | LZ4HC tuning    | 129 M (54.7%)      | Bit parity on resS-streamed cases; we already save 92 KB on the size side |
| Inline BC7 per-block partition drift  | BC7 long-tail probes | ~106 M (45.3%) | Closes long-tail BC7 partition/mode misses on the 330 inline bundles |
| Pattern A placeholder miss (>max src) | Headless-BC7 placeholder generalization (landed) | already collapsed  | 0 incremental — already on main |

The 23 byte-identical bundles are all in `28+142, no-.resS` — the
"inline small source" cohort that the BC7 long-tail probes are
actively closing. Expect this to grow as the BC7 probes land.
