# Mesh tangent 1-ULP residual — irreducible boundary noise

**Why it matters:** Computed mesh tangents are the last residual keeping some meshes from byte-identity. Most of the miss is a 1-ULP rounding difference on individual tangent components — the final f32 lands one unit-in-the-last-place off the reference. If a predictable rule or arithmetic ordering drove that rounding, it could be reproduced and the meshes closed.

**How it works:** This is a negative finding for the 1-ULP class. Tangents are computed only when a primitive lacks TANGENT and its material requires them (has a normal map); supplied-TANGENT meshes use a verbatim copy path and are already byte-exact. The computation reimplements the `Mesh.RecalculateTangents()` algorithm the reference corpus is generated with, whose native implementation is closed-source C++. Characterizing the exactly-1-ULP components shows the rounding direction is unbiased — diff sign, mantissa LSB parity, lane, value sign, and round-to-even target all come out at chance, so no predicate predicts which way a component rounds. Running many arithmetic variants through the full pipeline (FMA placements, reciprocal vs divide normalize, rsqrt, atan2 vs acos for the angle, pre-normalized edges) leaves the byte-identical mesh count unchanged and the bit count barely moving; the best variant nets only a few recovered components and zero whole meshes, which is the random churn expected when nudging an f64 value sitting near an f32 boundary. Variants that inject an extra f32 narrowing mid-chain regress, confirming the reference keeps double precision throughout and any premature narrowing is wrong.

The irreducible core: our f64 chain and the reference's native MSVC-compiled `RecalculateTangents` agree on the true tangent to far better than f32 precision and disagree only on which side of a rounding boundary a few components land, because the two execute a different exact f64 instruction sequence (notably the `acos`/`sqrt`/divide primitives — MSVC libm vs platform libm). Closing it would require reproducing that native instruction stream bit-for-bit, which is off-limits clean-room. A separate, larger structural class of meshes (errors well beyond 1 ULP) comes from UV-seam / multi-primitive-merge topology, where the reference recalculates tangents on the merged multi-submesh mesh while we compute per glTF primitive — a distinct, higher-yield target if pursued.
