# Morph-target residuals: AnimationClip float curves and SkinnedMeshRenderer blend-shape weights

**Why it matters:** Two rare singleton residuals — one on an AnimationClip, one on a SkinnedMeshRenderer — both trace to the same missing feature: glTF morph targets (Unity blend shapes). abgen-rs had no morph-target support in the animation pipeline at all, so two fields that should pull data from the glb were emitted unconditionally empty. Neither is numeric drift; each is a data-population gap.

**How it works:** The two cases are the two halves of morph-target support.

Animating the weights must populate the clip's float curves. A glTF `weights` animation channel stores its output target-major within each timestamp (all targets for t0, then all for t1, and so on). Unity de-interleaves this into one float curve per (node path, target index), with the attribute named `blendShape.<targetName>` taken from the mesh's extras target names (falling back to the Unity unnamed convention when absent). Each curve uses the scalar twin of the existing vector keyframe shape, and the same linear finite-difference slope baking already used for rotation/translation/scale transfers verbatim. The node path is the same encoding already produced for transform curves. Because this is a legacy clip, the generic bindings and motion-curve fields stay empty.

The static initial weight must populate the SkinnedMeshRenderer's blend-shape-weights field. This is the glTF `mesh.weights` array (distinct from the per-vertex skinning weights), which abgen-rs was not extracting at all. Plumbing it from the glb through the scene primitive into the SMR builder, and writing the field, closes the residual. A mesh with no morph targets yields an empty array, which is what the base prototype already produces, so the change is a no-op elsewhere. Note this field is independent of the mesh's blend-shape channel data: reconstructing the mesh shapes so blend shapes actually deform at runtime is a separate, larger feature and is not needed to close these parity residuals.
