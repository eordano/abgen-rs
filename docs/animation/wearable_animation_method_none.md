# Wearables convert with AnimationMethod.None (skeleton-only bundles)

**Why it matters:** Some avatar/emote entities are stored with `type:"wearable"` (legacy emote items) even though their glb contains an animation and an `Armature` root node. Unity's converter emits these as a tiny skeleton-only bundle — just the bone hierarchy, with no AnimationClip, no Animation component, and the `Armature` root collapsed into the hash-named bundle root. abgen-rs was instead emitting the full legacy Animation plus AnimationClip and a redundant `Armature` GameObject, producing bundles orders of magnitude too large and never byte-identical.

**How it works:** The converter's `GetAnimationMethod(isEmote, isWearable)` checks the wearable case first and returns `AnimationMethod.None` for it, before the emote check. So any entity whose type contains "wearable" — even a legacy emote stored under that type — converts with no animation method. `AnimationMethod.None` then propagates into the glTF importer with two consequences: no animation clips are read, so no AnimationClip or Animation objects are created; and because the scene then has no animation and a single root node, the importer collapses that single root and renames it to the bundle hash rather than wrapping it under a synthetic scene object, so the `Armature` name disappears. The result is a pure bone skeleton.

abgen-rs reproduces this in `src/builder.rs` with an `is_wearable` flag (computed from the entity type, threaded alongside the existing emote flag). For wearables it forces the "has animation" state to false — which both suppresses the legacy AnimationClip/Animation emission and flips the single-root collapse-and-rename behavior — and additionally gates the legacy animation branch on not being a wearable. This is fully recoverable clean-room because it depends only on the entity type, which abgen already has. It cannot regress other wearables: the converter strips animation from every wearable, so no wearable reference bundle contains animation objects to begin with.
