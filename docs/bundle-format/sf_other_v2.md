# Legacy-scene metadata TextAsset suppression

**Why it matters:** abgen-rs emitted a `metadata` TextAsset object into every scene bundle. The legacy (pre-v3, CIDv0) converter did not — only the v3+ converter (CIDv1 scenes) writes it. Emitting it unconditionally meant every legacy scene bundle carried one extra object, and that single extra object rippled through the whole SerializedFile: an extra type entry and typetree blob, an extra object-table row, an extra container entry, an extra inter-object gap, and the resulting shifts in metadata size and data offset. This was the single largest contributor to the otherwise-unexplained SerializedFile byte differences on legacy scenes.

**How it works:** A build option (defaulting to on, to preserve current behavior for modern CIDv1 entities) gates both the metadata TextAsset object and its matching container entry. Legacy entities, identified by their CIDv0 (`Qm`-prefix) identity, build with the option off, dropping the object and aligning the SerializedFile layout with the legacy converter. The default-on choice keeps the CIDv1 fixtures bit-identical.

This note also documents the decomposition tooling that made the diagnosis possible: it walks the SerializedFile metadata region by region (header, type table, typetree nodes and string buffers, object table, externals, data section, and inter-object padding), pairs regions across declaration-order differences by class and ordinal, and attributes the byte differences per bucket — letting an unexplained "other" diff be split into concrete causes.

Two residuals remain on legacy scenes. The first is a leftover orphan TextAsset type slot: even with the object suppressed, the type declaration can still be written if the base prototype carries it and unused types are not pruned, so the type slot must also be elided. The second is a legacy `.gltf` scene-primitive shape difference, where abgen-rs spawns a separate child GameObject and Transform for a primitive that the legacy converter collapses into the parent — gating that child-GameObject emission on the legacy entity version is the open follow-up.
