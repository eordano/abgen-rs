# Unity-editor probes: no-NORMAL mesh import and AnimationClip sub-asset PathIDs

**Why it matters:** Two behaviors that abgen-rs could only guess at were resolved by driving the live Unity editor against a real glb and capturing exactly what the importer produces. The first is how Unity fills in vertex normals when a glTF primitive has none; abgen-rs had been emitting a flat placeholder, which cannot match the reference's mesh bytes. The second is how Unity assigns the `LocalIdentifierInFile` (PathID) to AnimationClip sub-assets of an AnimatorController — a value that had resisted every attempt to derive it from the inputs.

**How it works (normals):** When a primitive lacks a NORMAL attribute, Unity does not reorder vertices. It computes area-weighted vertex normals on the post-import topology (after the x-axis flip and the triangle winding flip that abgen-rs already applies): for each triangle it takes the cross product of two edges — whose magnitude is proportional to the face area — accumulates that vector onto each of the triangle's three vertices, then normalizes. abgen-rs reproduces this in `src/gltf.rs`, re-deriving the flipped index list inside the normal block so it does not depend on later index-build ordering, and a test locks the formula at f32 precision against the captured probe data. One open residual: vertices whose accumulator cancels to zero (opposite-winding duplicate triangles) get an axis-aligned fallback whose exact rule Unity uses is not yet characterized.

**How it works (AnimationClip PathID):** This is a negative finding. The AnimationClip sub-asset PathID is not a function of any combination of parent GUID, clip name, or add order — two identical runs of the same input produce different PathIDs. Unity assigns it from process-local randomness on the first `AddObjectToAsset` call, then records it in the asset's `.meta` file; the deterministic-GUID fork rewrites only the GUID line and leaves that recorded PathID untouched, so every later import preserves whatever the original machine happened to roll. The production PathIDs therefore cannot be reproduced clean-room from the source glb and CID alone — they would require the original `.meta` files, which do not ship with the bundles. This line is closed as fundamentally blocked, not as a search that fell short.
