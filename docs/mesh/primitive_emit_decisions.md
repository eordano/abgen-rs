# Unified per-primitive emit decision tree

**Why it matters:** Each glTF primitive attached to a node's GameObject must produce exactly the set of Unity components prod produces — and the outcomes interact. Earlier single-rule attempts each fixed one case while regressing another: emitting a mesh-filter/renderer whenever a primitive has no material over-fires on skinned-collider parents (prod ships those Transform-only), and suppressing components on every collider node over-fires on the large majority of colliders that do get a mesh-filter plus collider. Picking the wrong rule trades one residual category for a larger one.

**How it works:** The outcome for the parent primitive is decided by four bits read directly from the glTF data, with no import-order replay or per-bundle overrides: whether the node name marks it a collider, whether it is skinned (has a skin plus joint/weight attributes), whether any primitive in the node's mesh has morph targets, and whether this is the node's parent primitive versus a child. A primitive becomes a SkinnedMeshRenderer if it is skinned or has morph targets.

The rules, in order: a collider parent that would become an SMR ships Transform-only (no renderer, though the mesh and material are still allocated because prod lists them in the preload table); otherwise an SMR-eligible primitive emits a SkinnedMeshRenderer (a morph-only mesh with no skin ships empty bones and a zero root bone, since the SMR is the component Unity uses to evaluate blend shapes); otherwise a collider emits a mesh-filter plus mesh-collider (allocating an orphan material when one is specified); otherwise it emits a mesh-filter plus mesh-renderer, falling back to the default scene material when none is specified.

The key insight is that the discriminating combination is collider AND (skinned OR morph): that interaction isolates the Transform-only cases cleanly. Morph targets, not the animation channel, drive the SMR choice — the same animation channel (e.g. rotation) appears on both renderer and SMR outcomes, so it cannot be the discriminator, whereas the mesh-level morph signal partitions them without error.
