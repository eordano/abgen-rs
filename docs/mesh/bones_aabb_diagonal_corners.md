# Mesh `m_BonesAABB` — diagonal-corner projection

**Why it matters:** Each bone's per-bone bounding box (`m_BonesAABB`) is serialized into the Mesh, so any deviation from the reference's exact values breaks byte-identical output. An earlier rule projected only the two points `base` and `base + morph_delta` through the bind pose, which undershot the true extremes whenever a morph delta had mixed-sign components and the bind pose applied a non-axis-aligned rotation. The resulting bone bounds were too small along some axes.

**How it works:** For each vertex weighted to a bone, abgen-rs first builds the mesh-local AABB over the vertex's base position and its position under every morph target it participates in, yielding two opposite corner points (elementwise min and max). It projects both of those corners through the bone's bind pose, then unions the per-axis min and max of the projected pair into the bone's running AABB. Because the bind-pose transform is affine, the AABB of the projected source box equals the AABB of the projected min/max corner pair — each output axis is an independent affine combination that is monotone in each input axis — so these two diagonal corners are sufficient to capture all extremes. This is what the earlier two-point rule missed: when a rotated, mixed-sign delta box is projected, the extreme projected values land on diagonal corners that are neither `base` nor `base + delta`. The rule was recovered by synthetic-input probing of a running Unity Editor and verified bit-exact across all finite bones and axes of a canonical falsifier mesh.
