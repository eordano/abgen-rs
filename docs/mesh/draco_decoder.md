# Draco mesh decoding

**Why it matters:** Some glTF inputs require `KHR_draco_mesh_compression`; without a decoder, abgen-rs early-returns "unsupported glTF extension" and cannot build those bundles at all. Decoding lets them build, but byte-identical parity with prod is a separate and harder question, because Unity imports draco meshes through a different path than plain glTF.

**How it works:** No pure-Rust draco mesh decoder exists, so abgen-rs vendors the C++-FFI `draco_decoder` crate (Google's draco via a cxx bridge) under `third_party/`, integrated through `src/draco.rs`. Two upstream patches were needed: the build script must probe both `lib`/`lib64` install dirs and use an absolute path (rustc's linker runs from a different working directory than the build script), and the public `MeshAttribute` must thread through draco's per-attribute `unique_id` so attributes can be mapped to the glTF extension's `attributes` entries reliably instead of relying on a fragile sort-by-id convention.

`src/draco.rs::materialize` runs in-place during glTF load: for each primitive carrying the extension it decodes the buffer, appends fresh indices and per-attribute byte streams, synthesizes the matching bufferViews and accessors, rewrites the primitive's attribute and index references, and strips the extension. Downstream the parser sees plain glTF; because draco accessors lack JSON min/max, the AABB falls back to the stream-scan path, matching Unity which recomputes the AABB from the decoded floats.

**Caveat — this is a coverage win, not a parity win.** The decoded float arrays should be deterministic, but Unity's draco import path does vertex dedup, reorder for cache locality, and channel-layout choices that abgen-rs does not model on the draco branch. So these bundles moved from "unbuildable" to "buildable but high-residual." They now produce output and can be validated end-to-end, but closing the residual requires a side-by-side comparison against the converter's own draco import path to attribute the gap between decoder-side quantization match and the converter's post-decode mesh passes.
