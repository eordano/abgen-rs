# Spec-gloss material workflow

**Why it matters:** Some glTF materials describe their surface with the legacy `KHR_materials_pbrSpecularGlossiness` extension instead of the standard metallic-roughness block. The converter's URP material import path honors that extension as an alternate base-color + specular workflow, but abgen-rs ignored the extension entirely — so for every spec-gloss material the PBR fields fell back to defaults and the emitted Material diverged from the reference.

**How it works:** When the extension is present, base color comes from the extension's diffuse factor and diffuse texture, not from the metallic-roughness `baseColorFactor`/`baseColorTexture` — abgen-rs drops the metal-rough block from base-color routing to mirror the reference. The specular-glossiness texture is bound to the `_SpecGlossMap` slot (threaded through the same cross-bundle PPtr, sampler-dedup, and texture-transform machinery as the other slots, and classified as a linear texture). The glossiness factor is mirrored into both `_Glossiness` and `_GlossMapScale`; the specular factor is written into `_SpecColor` as linear with alpha one (no gamma). The reference pins `_Metallic` to zero and uses its spec-gloss-tab smoothness default, and it lists `_SPECGLOSSMAP` among the invalid keywords (sorted) — abgen-rs reproduces all of these and skips the metal-rough emission branch. Crucially the reference ignores the metallic-roughness block *entirely* in spec-gloss mode even when the glTF still carries a `metallicRoughnessTexture`: that texture is not bound to `_MetallicGlossMap` and does not enable the `_METALLICSPECGLOSSMAP` keyword, so abgen-rs nulls the metal-rough texture reference when spec-gloss is active.

**Residual (not yet modelled):** The reference emits a `Texture2D` pair for *every* glb image, not just slot-bound ones. The orphaned metal-rough-block images (the diffuse/metalRough textures the spec-gloss workflow discards) still ship, but the reference compresses them as DXT5/BC3 (format 12) rather than the BC7 (format 25) it uses for slot-bound textures. abgen-rs only emits slot-bound images and has no BC3 encoder, so a spec-gloss bundle that carries unbound metal-rough textures is short those pairs. Closing it needs an unbound-image emission path plus a BC3 encoder.
