# BC7 encoder preset for standalone textures (Basic vs Slow)

**Why it matters:** The compressed BC7 image data for standalone texture bundles was diverging from Unity at the per-block level on Windows and mac builds, even though the metadata and dimensions matched. The pure-Rust BC7 encoder was correct, but it was running the wrong quality preset for this code path, leaving the pixel bytes one or more blocks off from production.

**How it works:** Unity routes textures through two different importers with two different bc7e presets. In-glb textures (driven through the custom glTF importer) use the `slow` preset, which abgen-rs already matched byte-for-byte. Standalone texture bundles (PNG/JPG bundled by `ImportTextures`) go through a `TextureImporter` whose default platform override on the desktop targets is the `basic` preset at perceptual mode. Selecting the BC7 profile per call site — `slow` for the in-glb path, `basic` for the standalone path — makes the encoder match production. Confirmation came from re-encoding production blocks against every bc7e preset and observing that `basic`/perceptual wins across the entire corpus, reaching essentially total per-block agreement on clean power-of-two downscales. The residual after switching presets is two separate, narrower issues: per-block tie-break noise between the pure-Rust port and bc7e on non-clean geometries, and a resize divergence on upscale / near-identity targets — both distinct from preset selection.
