# The BC7 "tie-break" residual is alpha-bleed, not encoder ambiguity

**Why it matters:** A large standalone-texture residual was long attributed to BC7 encoder tie-breaks — presumed differences between abgen-rs's pure-Rust port and the bc7e ISPC encoder's refinement scheduling (bc7e is the open-source Intel/GameTechDev encoder the converter invokes). If true, it would have justified a deep encoder-internals workstream. This note re-diagnoses the residual and shows that diagnosis was wrong, redirecting the effort to the actual cause.

**How it works:** The pure-Rust BC7 port is bit-exact to bc7e ISPC at its basic preset, and no bc7e preset or parameter tweak can reach the converter reference's bytes from abgen-rs's input pixels — every mechanical encoder knob either makes the residual worse or does nothing. The divergence is upstream of BC7, in the source pixels. the converter's `TextureImporter.alphaIsTransparency` preprocessing bleeds RGB from nearby opaque pixels into fully-transparent (alpha-zero) regions before encoding, whereas abgen-rs feeds the canonical transparent-black source through unchanged. Every differing block contains at least one transparent pixel, and every fully-opaque block already matches byte-for-byte; once the input pixels differ, the resulting mode, partition, endpoints, and indices cascade apart. The fix therefore lives in source-pixel preprocessing (the alpha-bleed dilation pass), not the encoder. Only once that preprocessing matches the reference will any genuine encoder tie-break residual become visible above this much larger signal.
