# BC7 mode-1 vs mode-6 partition probe

**Why it matters:** On real reference bundles, the contested BC7 decision is mode-1 (two-subset opaque) versus mode-6 (single-subset RGBA). When abgen-rs picks a different mode than the reference, the two encodings share no bit layout, so the entire block diverges. Understanding when bc7e (the open-source Intel/GameTechDev encoder the converter invokes) chooses each mode, and which partitions it prefers, is needed to steer abgen-rs's mode selection toward parity.

**How it works:** Characterizing the reference's per-block choice against pixel features (rather than against abgen-rs output) reveals the discriminators. The overwhelming majority of prod-bundle blocks are flat, near-uniform regions where mode-5 wins trivially; mode-1 and mode-6 only compete on the textured remainder. Mode-1 in this corpus is always fully opaque — the reference never selects it on alpha-varying blocks — so the mode-1 versus mode-6 contest is meaningful only on opaque blocks, where RGB variance is the separator: high color variance predicts mode-1 with good precision, low variance favours mode-6. Among mode-1 blocks the partition choice is highly concentrated on the two "clean half" partitions (vertical and horizontal splits), consistent with grid-aligned terrain and atlas content; a naive best-mean-split heuristic lands the reference's exact partition a large fraction of the time and within its top-few candidates almost always. This is a characterization probe: it maps the reference's behaviour conditional on pixel features but does not pair reference output against abgen-rs output, so it bounds the available parity gains without measuring them directly.
