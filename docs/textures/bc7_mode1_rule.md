# BC7 mode-1 vs mode-3: the high-variance gate

**Why it matters:** A large share of blocks where Unity chose BC7 mode 1 came out of abgen-rs as mode 3 instead. Both are 2-subset opaque modes, but mode 3 trades index resolution for endpoint precision (fewer index levels, more endpoint bits), while mode 1 trades the other way (more index levels, shared-pbit endpoints). On high-variance gradient blocks, mode 3's lower quantization noise lets it win the raw SSE race, so a pure error comparison picks mode 3. Unity overrides this and reserves mode 1 for the gradient case, so the encoder's mode pick diverged on exactly the blocks where mode 1's gradient resolution matters most.

**How it works:** The two modes separate cleanly on per-block RGB variance. Unity's mode-3 population is the low-variance, two-tone regime; its mode-1 population is the high-variance gradient regime, and the two distributions barely overlap. abgen-rs applies a small SSE penalty (roughly an eighth) to mode-3 trials when a block's RGB variance exceeds a fixed threshold, which flips the marginal blocks toward mode 1 without disturbing blocks where mode 3 wins decisively.

This is the symmetric counterpart to the landed low-variance rule that biases mode 1 toward mode 6 below a separate, lower threshold. The two thresholds leave a deliberate neutral band between them where both biases are inert, so they target disjoint mismatch populations and never conflict. The rule deliberately does not touch partition selection: it only changes whether mode 3 wins the comparison, sidestepping the partition-shortlist interactions that broke earlier attempts. The remaining same-partition endpoint residual — dominated by partition 0 — is a separate endpoint-refinement problem, not addressed here.
