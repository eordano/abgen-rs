# Standalone textures inherit the quality-100 BC7 profile when a model references them

**Why it matters:** The converter encodes most standalone texture bundles with the BC7 encoder's default quality level (the Basic profile here), but a subset of them come out encoded at the maximum quality level (the Slow profile). Encoding those with Basic produced bundles whose 4×4 blocks systematically used two-subset modes where the reference used three-subset modes, and on hard color edges the restricted mode set occasionally picked an objectively wrong partition — single texels up to 197/255 levels away from the source where the reference reproduced the source almost exactly.

**How it works:** The converter imports every downloadable texture with high-quality compression but leaves the compression-quality knob at its default. When a glTF or GLB later references that same texture file as an external image, the model importer re-imports the texture file with the quality knob raised to 100, and that importer state sticks to the asset — so the *standalone bundle* built from the same file inherits the maximum-quality encode. Unity maps the raised knob to the BC7 encoder's slowest level, which evaluates the full mode set (including the three-subset modes 0 and 2 and the seven-bit-endpoint mode 3) instead of the perceptual default's restricted set.

abgen already computed exactly this predicate for the streaming gate: a standalone texture is `model_referenced` when a model in the same entity resolves one of its image URIs to that texture's content hash, with the converter's quirk that a model whose URI list does not fully resolve contributes no references at all. The fix reuses that flag to pick the encoder profile: model-referenced standalone textures encode with the Slow profile, everything else stays Basic. The same machinery already drives the linear color-space override for metallic-roughness and normal-map usage.

**Evidence:** Switching the profile on the `model_referenced` predicate flipped 53 more bundles to byte-identical with zero regressions across the validation and world corpora — including the bound-but-already-identical bundles, which confirms the predicate matches the converter's actual behavior rather than a looser "appears in some image list" rule. On the previously affected pairs the decoded-pixel picture improved across the board: the count of pairs whose every pixel decodes within one 8-bit level went from 2 to 85, the worst single-texel error fell from 197 to 132 levels, and the median worst-texture mean error dropped 64×. The residual is the usual encoder float-order wall, now operating under the correct profile.

**Diagnosis trail:** the asymmetric mode-flip signature (ours emitting only modes 1 and 6 where the reference used 0, 2, and 3) pointed at a profile mismatch; an eight-way profile/perceptual/color-space sweep against the reference payload identified Slow-perceptual-sRGB as the reference's configuration for the affected texture; binding analysis over entity manifests separated the affected population (material-bound textures) from the unaffected one (pure standalone textures, which stay Basic and match byte-exactly).
