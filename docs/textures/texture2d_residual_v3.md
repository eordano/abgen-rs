# Streaming standalone BC7 textures regardless of size

**Why it matters:** Standalone texture bundles whose BC7 image is referenced by a sibling glb model must have their pixel data streamed into the `.resS` sidecar, exactly as Unity's production converter does. abgen-rs was streaming only the 512×512 subset and inlining every other size, so larger standalone textures diverged on `m_StreamData.path`, `m_StreamData.size`, `m_IsReadable`, and the inlined `image data` itself — blocking byte-identical output for every standalone BC7 above that size.

**How it works:** Unity's `ImportTextures` path streams every cross-bundle-referenced BC7 standalone texture, at any resolved resolution. The correct gate is simply "this is a BC7 texture referenced by a sibling glb", expressed by the `model_referenced` flag that abgen-rs already plumbs through from the glb URI-collection pass. The earlier extra `512×512` size predicate was a relic of an old Linux build target that capped the max texture size at 512; on Windows and mac, where the cap is larger, it wrongly excluded the bigger textures. Dropping the size predicate and gating purely on `model_referenced && format == BC7` reproduces Unity's partition exactly: every referenced standalone BC7 streams, every unreferenced one stays inline.
