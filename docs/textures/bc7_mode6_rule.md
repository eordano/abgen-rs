# BC7 mode-6 parity rules

**Why it matters:** Mode-6 (single-subset RGBA, full-precision indices) is the largest single-mode source of BC7 byte divergence from the reference. Misses fall into two classes: blocks where the reference picks mode-6 but abgen-rs picks a different mode (chiefly mode-5), and blocks where both sides pick mode-6 but disagree on the encoded bits. Each costs the full 16-byte block, so closing them is required for byte-identical texture output.

**How it works:** Several independent tie-break behaviours of bc7e (the open-source Intel/GameTechDev encoder the converter invokes — see NOTICES.md) were recovered by black-box analysis of the reference's mode-6 blocks. When endpoint error is otherwise tied or near-tied, the reference exhibits a clear preference for setting both p-bits to 1, and the low p-bit additionally tends to match the parity (LSB) of the block's per-channel pixel minimum — a natural consequence of least-squares snapping the low endpoint while preserving the pixel-min low bit. abgen-rs reproduces these as small epsilon-scaled biases in the p-bit selection step, affecting only modes that carry explicit p-bits. Separately, the reference favours mode-6 over mode-5 on opaque, low-color-variance blocks whose alpha warrants mode-6's index precision; abgen-rs strengthens its mode-6-over-mode-5 preference, gated on block features so the dominant mode-5 path is not regressed. Selector ties (a pixel exactly midway between two interpolated palette colors) round toward the higher selector. The mode-6 bit-packing and endpoint-swap convention were verified already reference-compatible, so they are left unchanged.
