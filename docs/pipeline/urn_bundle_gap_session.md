# Collection-URN bundle-count gap is content drift (negative finding)

**Why it matters:** Building the base-avatars collection by URN emits fewer bundles than the converter reference for the same collection. The proposed explanation was that the extra reference bundles are glb-embedded textures — image URIs inside a glb that the lambdas collections/wearables DTO does not list as top-level content — and that the converter should parse each glb's image URIs, resolve them against the wearable content, and emit a standalone-texture bundle per resolved image. If correct, this would be a concrete missing emit rule blocking a full bundle set match.

**How it works:** This is a negative finding — the gap is content-version drift between the reference and the live lambdas collection content, not a missing emit rule. abgen-rs emits exactly the DTO content set, deduped: one bundle per glb and per listed image. The embedded-texture hypothesis is false because every glb-embedded image URI already appears as a top-level content item, so there is no omitted-embedded-texture class. The bundles present only in the reference are stale hashes that were the collection's content when the reference was generated and have since been redeployed; they no longer appear in the current DTO. The bundles present only in abgen-rs are the corresponding newer content. Many wearables drifted between reference-generation time and now. No converter change closes this; the only remedies are operational — regenerate the reference against current content, or pin and replay the original collection DTO snapshot through the corpus driver. Byte-parity tooling already intersects to the matching bundle names, so the drift does not pollute parity numbers; the tractable work is the BC7 and mesh classes in the matched bundles.

A second independent audit re-checked this finding and confirmed it: a clean-room reading of the upstream converter shows it builds its content mapping purely from the wearable representations' content lists and never enumerates embedded image URIs to create extra bundles, so abgen-rs mirrors it exactly. Matching the reference count would require feeding the converter the same pinned content snapshot the reference used — a data-provisioning task, not a source change.
